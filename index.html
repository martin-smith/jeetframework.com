Basic

It's that easy. But look carefully at the gutter between these two elements and, say, the gutters between the navigation buttons. One of these things is not like the other! It's because we're using percentage-based gutters. So 2% of a 1000px container is going to be bigger than 2% of a 100px container. It works, but it effectively defeats the whole purpose of using grids since they're meant to be uniform.

How can we fix this?

Nested

If we pass the ratio of the parent container (<div class="result"> is the container of these elements and it's set to col(1/2) - go ahead and inspect) then we end up with consistently sized nested gutters.

You need to do this with every parent that is using col() or span() so if you're doing some crazy-deep nesting, you may end up with something that looks like this: col(1/2 1/3 1/2 1/4). Don't be scared. Just step through your code and you'll get the hang of it. Then you'll be the coolest kid on the block with your nice, percentage-based, consistent, gutters.

The only other grid that attempts to do this correctly is Scott Kellum's Singularity.gs. Singularity's syntax isn't my cup of tea, but it really is a brilliant system and the only other system that provides incredibly flexible ratio/percentage-based columns while maintaining a consistent gutter.

Props aside, we've worked hard to make Jeet 3's syntax very intuitive.

Offsets

Want to put a bit more space between items? Leave bit off of your col() (or span()) elements, then pass the offset as a param to one of your elements.

You can also use a negative value for your offset to have it offset in the opposite direction.

Centering

Jeet comes with the center() mixin which accepts a max-width as it's first argument, and left/right padding as it's second. You can use it on anything, but it works best on wrapper elements. It's simple, effective, and useful.

One of my favorite uses for center() is to set a full-width background, then make the immediate child <div> of that element center()'d to center content.
